# 予定の手動完了デザイン見直し（デザイン例の提示）

## 目的・背景

Canvas 課題の手動完了チェック（[canvas-assignment-completion.md](archive/canvas-assignment-completion.md)）で
iOS の締切行に丸チェック UI を付けたが、以下の課題がある。

1. **チェックできる行とできない行が混在** — uid のある Canvas 由来だけ左端に丸チェックが付き、
   カレンダー由来の締切には付かない。行頭が揃わず、なぜ操作できないのかも見た目で分からない
2. **タップ領域が小さい** — アイコンだけの小さなボタンで押しにくく、隣の DuePill との誤タップもある
3. **完了済みが一覧に残り続ける** — 打ち消し線のまま「今後の締切」に並び、リストが伸びる
4. **完了時のフィードバックが乏しい** — アニメーション・取り消し導線がない
5. 同じ deadlineRow 実装が HomeView / CalendarTabView に重複しており、デザイン変更の障害になる

## 対応方針

本タスクは**デザイン検討**（実装は次タスク）。以下を行う。

- Step 1: 現状 UI の課題整理（上記）
- Step 2: デザイン案を複数作成し、モックアップ（HTML アーティファクト）で比較提示する
  - 案A: チェックサークル改良（全行で行頭を揃える + 完了済みを折りたたみ）
  - 案B: スワイプ完了（iOS 標準パターン。行本体からチェックを消してクリーンに）
  - 案C: DuePill 統合（残日数ピル自体をトグルにし「済」ピル化。状態を形で示す既存方針に合致）
  - 案D: 行全体タップ + 完了行の自動整列 + Undo
- Step 3: ユーザーが案を選んだら実装タスクを TODO.md に起こす

## 影響範囲

- iOS: `HomeView.swift` / `CalendarTabView.swift` の deadlineRow（採用案によっては共通コンポーネント化）
- 管理画面 `/admin/calendar-info` のチェックボックスは対象外（必要なら別タスク）
- backend API（POST /deadlines/complete）は変更しない

## テスト方針

- デザイン検討段階のためコード変更なし。採用案決定後の実装タスクで
  シミュレータビルド + 実機確認（チェック操作 → サーバ保存 → 再取得で状態維持）を行う

## 結論（2026-07-17）

4 案を[モックアップ](https://claude.ai/code/artifact/c1f28558-2d13-4767-949e-0d347050c199)で比較した結果、
**既存デザイン（Canvas 由来のみ丸チェック + 打ち消し線）を維持**することに決定。実装変更は行わない。
将来リストの見通しが悪くなった場合は、案A の「完了済み折りたたみ」だけ独立して導入する選択肢がある。
