# 管理画面の整理: ダッシュボードを必要最低限にする

## 目的・背景

現状の管理画面（`backend/assets/admin.html`）はダッシュボード 1 ページに
「stats / 実行ボタン / 最新ブリーフィング / 収集カレンダー選択 / 収集設定 /
push ログ / コレクタ実行 / 登録デバイス / 実行ログ末尾」が全部詰まっていて縦に長い。
毎日見るのは stats と最新ブリーフィングだけなので、ダッシュボードを必要最低限にし、
運用ログ系と設定系を別ページに分ける。

## 対応方針

単一 HTML 内のセクション出し分け（既存の `showPage()` パターン）を維持し、
ページを 4 → 6 に増やして再配置する。**backend（API・DB）は不変**。

| ページ | 内容 |
| --- | --- |
| ダッシュボード | stats 5 枚 + 「今すぐ実行」ボタン + 最新ブリーフィング本文（現状から縮小） |
| カレンダー | 既存のまま（今後の予定 / 今後の締切） |
| 買い物リスト | 既存のまま |
| AI 利用状況 | 既存のまま |
| システム（新設） | 直近の push / 直近のコレクタ実行 / 登録デバイス / 最新ログ末尾 |
| 設定（新設） | 収集カレンダー選択 + Canvas 先読み日数 |

- データ取得は現行どおり `/admin/status` 1 本。ダッシュボードとシステムの両セクションは
  DOM 上に常に存在する（hidden で出し分け）ため、`refresh()` が両方を埋める構造は変えない。
  ジョブ実行中の 3 秒ポーリングもそのまま。
- 設定ページの読み込み（`refreshSettings()`）は従来どおり認証時 1 回 + ページ表示時に実行。

## 影響範囲

- `backend/assets/admin.html` のみ（セクション移動 + サイドバー項目追加 + ページ切替登録）
- backend の TypeScript / API / DB / iOS は不変

## テスト方針

- `node --check` で script 部の構文確認（既存コミットと同じ手法）
- `./run-admin.sh` 相当でローカル API サーバを起動し、ブラウザ（または curl + 目視）で
  各ページの表示・切り替え・既存機能（ブリーフィング実行 / カレンダー保存 / 設定保存）を確認
