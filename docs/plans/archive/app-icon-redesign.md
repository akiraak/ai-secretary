# アプリアイコンの変更

## 目的・背景

- 現在 `ios/AISecretary/Assets.xcassets/AppIcon.appiconset` には画像が未設定で、ホーム画面にはデフォルトの無地アイコンが表示されている
- [ios-app-screens.md](../../specs/ios-app-screens.md) の未確定事項にも「アプリアイコンの正式デザイン（モックはサンライズの暫定マーク）」が残っている
- アプリの顔となる正式アイコンを決めて設定する

## 対応方針

### Step 1: 方向性を決める

- 候補となるデザイン方向を SVG モックとして数案作成し、ブラウザでプレビューして比較する
- 候補の軸:
  - A. サンライズ（朝焼け）— 朝 07:00 ブリーフィングというコアコンセプトに合致。モックの暫定マークとも連続性がある。アクセントカラー（アンバー）とも調和
  - B. 秘書モチーフ（チェックリスト / ノート）— 「タスクを整えてくれる」機能を直接表現
  - C. モノグラム（「秘」の文字）— タイポグラフィで簡潔・視認性高
  - D. マスコット（ロボット秘書の顔）— 親しみやすさ重視
- ユーザーが方向を選定 → 本ファイルに決定を記録

### Step 2: デザイン確定・アセット生成

- 決定した方向: **ドラえもんを彷彿とさせる抽象アイコン**（キャラクターの複製ではなく、青地 + 白の丸 + 赤い帯 + 黄色い鈴という配色・構図の抽象オマージュ。ヒゲ・鼻など直接的な顔要素は入れない）
- 確定の進め方（2026-07-17 ユーザー指定）: **候補案をドキュメント（HTML プレビュー）に並べて提示 → ユーザーが選択 → 確定するまで何ラウンドでも繰り返す**
- 確定後、1024x1024 PNG を CoreGraphics で描画する Swift スクリプト（`ios/scripts/generate-app-icon.swift`）に反映し、再生成可能な形でリポジトリに残す

### Step 3: 組み込み・検証

- `AppIcon.appiconset` に 1024x1024 PNG を配置し `Contents.json` を更新
- iOS 18+ の Dark / Tinted 対応は必要になったら検討（まずは single size のみ）
- `xcodegen generate` → シミュレータビルドでアイコン表示を確認、実機は `./run-ios-device.sh` で確認
- [ios-app-screens.md](../../specs/ios-app-screens.md) の未確定事項から該当行を削除

## 影響範囲

- `ios/AISecretary/Assets.xcassets/AppIcon.appiconset/` のみ（コード・backend は不変）
- アイコン生成スクリプトを追加する場合は `ios/` 配下に置く

## テスト方針

- シミュレータビルド成功 + ホーム画面でのアイコン表示確認
- 実機インストールでの表示確認（ライト / ダーク背景での見え方）

## 決定記録

- 方向性: 2026-07-17 に一度「ドラえもん風の抽象アイコン」で進めたが、同日ユーザーが白紙に戻して再検討
- **2026-07-18 デザイン確定**: ユーザーが秘書キャラクター画像（`/Users/akiraak/Downloads/secretary.png`、1254x1254、白地にコーラルレッド #F85D67 の女性シルエット）を提供し、これを正式アイコンとする
  - `sips` で 1024x1024 に縮小して `AppIcon.appiconset/AppIcon.png` に配置（`sips -z 1024 1024 secretary.png --out AppIcon.png`）
  - 旧候補用の描画スクリプト `ios/scripts/generate-app-icon.swift`（青グラデ + 白ドーム + 赤帯 + 鈴）は不採用となったため削除
  - 同画像はアプリ全体の配色変更の基準にもなった（[app-theme-coral.md](app-theme-coral.md)）
