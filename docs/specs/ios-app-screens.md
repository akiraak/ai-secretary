## レイアウト決定（2026-07-14）

メイン画面のレイアウトは **案A「統合フィード（優先順1画面）」に決定**（ユーザー選択）。
1画面を上から緊急順にスクロールする。案B（2タブ）・案C（ダッシュボード）は比較の上で見送り、
必要になれば合わせ技として後で寄せる。

これに伴い **GitHub の作業把握を MVP のメイン画面に取り込む**（下記「よく使う機能」）。
元計画では GitHub は Phase 5 だったが前倒しする。

### よく使う機能（この設計が満たすもの）

1. 朝に **今日の作業と直近の締切** を確認する
2. **GitHub の前日までの作業** と **今後の作業（各リポジトリの `TODO.md`）** を確認する

## スコープ

MVP は **4 画面**。初回の通知許可 → 毎朝の受け取り → 表示 → 設定 を通す。
ウィジェット / Live Activities / 週表示 / チャットは後フェーズ（[spec Phase 3+](app-features.md)）。

## 画面

### 1. オンボーディング（初回起動）
- 目的: 通知を許可し、デバイストークンを backend（`POST /devices`）に登録する
- 内容: アプリの説明を「カレンダー / Canvas 締切 / 要対応メール」の3点に絞る
- ボタン: 「通知を許可して始める」/「あとで」
- 許可が下りたら APNs のデバイストークンを取得 → backend に送信

### 2. ロック画面通知（毎朝 07:00 PT）
- **主役のチャネル。** v1 は要点をフル本文で push（[ペイロード方式1](../plans/mvp-morning-briefing.md)）
- 文面例: 「タイムスケジュールの予定はなし。締切が2日後、要対応メール2件。Google One が7/19に終了します。」
- タップでブリーフィング画面（3）へ

### 3. 今日のブリーフィング（メイン画面）＝ 案A 統合フィード
- 構成: **LLM が書いた要約（上）＋ 緊急順のドメイン別セクション（下）を1画面スクロール**
- 大タイトル = 日付、サブ = あいさつ
- 「今日のまとめ」カード: LLM の日本語プロース要約（締切・GitHub・タスクを1〜2文で）
- セクション（緊急順、iOS グループ化リスト）:
  - **締切が近い**（Canvas iCal + Google カレンダー）: 残日数を赤ピル（例「あと2日」）
  - **今日やる**（各リポジトリの `TODO.md` から選抜）: 済ワンタップ＋リポジトリタグ
  - **要対応**（Gmail トリアージ）: 済ワンタップ＋期限ピル
  - **昨日の GitHub**（gh CLI: commits / PR）: リポジトリ別のコミット数
  - **次の作業**（`TODO.md` の残タスク、任意で折りたたみ）
- フッター: 「07:00 に自動更新 / 更新↻」（プル）
- 表示の作法: リポジトリ名は等幅・開発系は青、締切は赤ピル、TODO は済チェック（書き戻しは元 `TODO.md`）

### 4. 設定
- 通知: ブリーフィング ON/OFF、配信時刻（既定 7:00 AM）、タイムゾーン（Seattle）、言語（日本語）
- 連携ソース: Google カレンダー / Gmail / Canvas(iCal) の接続状態
- トリアージ: 要対応の判定基準、自動配信メールの除外トグル
- **原則**: マスターデータの編集は持たない。設定は最小限（[spec 2-5](app-features.md)）

## デザイン方針

- **配色**: 夜明けの光をモチーフ。AIの定番（クリーム×テラコッタ）を避け、クールなスレート地 × 琥珀アクセント。締切=赤 / 完了=緑 のセマンティックカラーはアクセントと分離
- **タイポ**: 日本語を含むため Web フォントは使わず **iOS システムスタック（SF Pro / Hiragino）** を意図的に採用。実機と同じ見え方。階層はウェイト・サイズ・数字の等幅（tabular-nums）で作る
- **情報設計**: 要約を上・詳細を下。状態を形で（残日数ピル / 済チェック / 接続バッジ）。締切の逼迫は色で一目化
- **ライト / ダーク両対応**

## この設計から実装 Step 7 への含意

MVP の iOS アプリ（[Step 7](../plans/mvp-morning-briefing.md)）が持つ画面はこの4つ。
バックエンド API との対応:
- オンボーディング → `POST /devices`
- ブリーフィング表示・更新↻ → `GET /briefings/latest`
- 通知本文 → APNs ペイロード（v1 フル内容）
